## 系統需求

| 項目 | 需求 |
|---|---|
| 作業系統 | macOS 12 Monterey 以上 |
| Python | 3.13（建議）|
| 套件管理 | Homebrew（安裝系統依賴）|
| 磁碟空間 | large-v3-turbo 約 455 MB；large-v3 約 3 GB |
| 麥克風 | 內建或外接麥克風皆可 |

## 安裝步驟

### 步驟一：安裝系統依賴

開啟終端機，執行：

```bash
brew install portaudio ffmpeg
```

- **portaudio**：麥克風錄音驅動
- **ffmpeg**：音訊處理工具

### 步驟二：建立虛擬環境

```bash
cd /path/to/project
python3 -m venv venv
source venv/bin/activate
```

### 步驟三：安裝 Python 套件

```bash
pip install -r requirements.txt
```

> 初次安裝約需 2–5 分鐘（依網速而定）。

---

## 啟動 App

```bash
venv/bin/python3 main.py
```

**首次啟動注意事項：**

1. App 自動在背景下載 Whisper 模型（預設 `large-v3-turbo`，約 455 MB），狀態列顯示「模型載入中…」。
2. 下載完成後，狀態列變為綠點 + 「模型已就緒」，即可開始錄音。
3. 若出現「需要輔助使用權限」彈窗，請依指示開啟（詳見[全域快捷鍵](#全域快捷鍵)章節）。

## 錄音操作

### 方式一：點擊按鈕

| 步驟 | 操作 | 按鈕狀態 |
|---|---|---|
| 1 | 點擊綠色「🎤 點擊錄音」按鈕 | 變為**紅色脈衝**「● 錄音中…」|
| 2 | 對麥克風說話 | 波形動畫即時更新，計時器開始運作 |
| 3 | 點擊紅色按鈕停止 | 變為**橘色**「⠋ 轉錄中…」spinner 動畫 |
| 4 | 等待轉錄完成 | 回到綠色；文字顯示在結果區 |

### 方式二：全域快捷鍵（Push-to-Talk）

**按住** ⌘⌥R → 對麥克風說話 → **放開** → 自動開始轉錄

> 此方式可在任何 App 前景時使用，無需切換回此視窗。自動貼上功能也依賴此方式觸發。

### 按鈕顏色說明

| 顏色 | 狀態 | 說明 |
|---|---|---|
| 綠色 | 待機（IDLE）| 就緒，可開始錄音 |
| 紅色脈衝 | 錄音中（RECORDING）| 正在收音，計時器運行中 |
| 橘色 + spinner | 轉錄中（PROCESSING）| Whisper 模型分析音訊，請稍候 |

## 自動貼上

這是本版本最核心的新功能：**說話 → 轉錄 → 自動貼回原本的 App**，無需手動切換視窗或複製貼上。

### 啟用方式

動作列中的「⌨ 自動貼上」按鈕：
- **紫藍色（啟用）**：每次轉錄完成後自動貼入目標 App
- **灰色（停用）**：僅顯示轉錄結果，不自動貼上

### 運作流程

```
1. 確認「⌨ 自動貼上」已啟用（按鈕為紫藍色）
2. 切換到目標 App（如 LINE、Notion、Word 等），游標點在輸入框
3. 按住全域快捷鍵 ⌘⌥R
        ↓ App 在按下瞬間記錄「目標 App 名稱」
4. 說話
5. 放開快捷鍵
        ↓ Whisper 辨識語音
6. 辨識完成後，自動切換回目標 App 並注入文字
```

### 使用注意事項

- 必須使用**全域快捷鍵**（⌘⌥R）才能正確捕捉目標 App；直接點擊 App 內按鈕無法捕捉外部視窗。
- 目標 App 必須有**輸入框且游標在焦點位置**，否則貼上位置不正確。
- 需要 macOS **輔助使用（Accessibility）** 權限，才能模擬 ⌘V 按鍵。

### 貼上目標提示

按下快捷鍵後，錄音按鈕下方會出現：

```
⌨ → LINE
```

表示轉錄完成後文字會自動貼入 LINE。若顯示空白，代表無法偵測目標 App（通常是缺少輔助使用權限）。

## 全域快捷鍵

### 預設快捷鍵

**⌘ + ⌥ + R**（Command + Option + R）

### 操作方式

- **按住** 快捷鍵 → 開始錄音
- **放開** 快捷鍵 → 自動停止並開始轉錄

### macOS 輔助使用權限

全域快捷鍵與自動貼上功能都需要「輔助使用（Accessibility）」權限。

**開啟步驟：**

1. 前往：**系統設定 → 隱私權與安全性 → 輔助使用**
2. 點擊「+」加入 **Terminal**（或執行此 App 的 python3 程式）
3. 確認右側切換開關為**開啟**狀態
4. **重新啟動** App

> 若未開啟此權限，App 視窗內的按鈕錄音仍可使用，但全域快捷鍵與自動貼上功能將無效。

### 自訂快捷鍵

1. 點擊「⚙ 設定」→「快捷鍵」區塊
2. 點擊「重新綁定」
3. 在彈出視窗中**按下並放開**你想要的按鍵組合
4. 確認顯示正確後點擊「確認套用」
5. 點擊「儲存」

> **注意：** 避免使用 `⌘⇧Space`，此組合在某些 macOS 版本會與系統功能衝突，App 啟動時會自動重置。

## 轉錄結果操作

### 複製到剪貼簿

點擊「📋 複製」→ 全部文字複製到系統剪貼簿。

### 儲存為文字檔

點擊「💾 存檔」→ 選擇儲存路徑與檔名 → 存成 `.txt` 格式。

### 清除結果

點擊「清除」→ 清空轉錄文字區。

### 多次錄音追加

預設開啟「追加模式」：每次新的錄音結果**附加**到現有文字下方，並以分隔線區分。可在設定中關閉，改為每次覆蓋。

### 結果標題資訊

轉錄完成後，結果區標題顯示：

```
📝  轉錄結果  (15s · ZH · large-v3-turbo)
```

- `15s` — 錄音時長
- `ZH` — 偵測到的語言（ZH=中文、EN=英文 等）
- `large-v3-turbo` — 使用的模型

## 設定面板

點擊「⚙ 設定」開啟設定視窗。

### 語音辨識

| 設定 | 說明 |
|---|---|
| 模型大小 | 選擇 Whisper 模型（詳見[模型選擇指南](#模型選擇指南)）|
| 辨識語言 | 指定語言或選「自動偵測」（支援中、英、日、韓、西、法）|

### 快捷鍵

| 設定 | 說明 |
|---|---|
| 全域快捷鍵 | 顯示目前設定的按鍵組合 |
| 重新綁定 | 點擊後進入偵測模式，按下新組合放開即完成 |

### 輸出

| 設定 | 預設 | 說明 |
|---|---|---|
| 每次錄音追加到結果 | 開啟 | 新結果附加到下方；關閉後每次覆蓋 |
| 轉錄後自動複製 | 關閉 | 每次轉錄完成後自動複製文字到剪貼簿 |
| 語音轉文字後自動貼上 | 開啟 | 轉錄完成後自動貼入偵測到的前景 App |

### 關於

顯示模型快取路徑與設定檔路徑，可點擊「開啟設定資料夾」在 Finder 中開啟。

- **設定檔位置**：`~/.whisper_app/config.json`
- **模型快取位置**：`~/.cache/huggingface/hub/`

## 模型選擇指南

| 模型 | 大小 | 速度 | 精準度 | 建議用途 |
|---|---|---|---|---|
| `tiny` | 39M | ⚡⚡⚡⚡⚡ | ★★☆☆☆ | 快速英文速記、測試用 |
| `base` | 74M | ⚡⚡⚡⚡ | ★★★☆☆ | 輕量日常使用 |
| `small` | 244M | ⚡⚡⚡ | ★★★★☆ | 中文辨識效果佳 |
| `medium` | 769M | ⚡⚡ | ★★★★★ | 高精準度，需 8GB+ 記憶體 |
| `large-v3-turbo` | 809M | ⚡⚡⚡ | ★★★★★ | **預設推薦**：中英混合最佳，Metal GPU 加速 |
| `large-v3` | 1.55G | ⚡ | ★★★★★ | 最高精準度，需 16GB+ 記憶體 |

**切換模型：** 頂部工具列下拉選單直接選擇，或進入設定面板。切換後首次轉錄會自動下載新模型。

**中英混合語音建議：** 使用預設的 `large-v3-turbo`，此模型專為中英夾雜語音最佳化，配合 Apple Silicon Metal GPU 加速，速度與精準度兼顧。

## Ollama 整合（進階）

Ollama 功能可讓本地 AI 語言模型自動潤飾轉錄文字（修正錯字、調整語氣、重新斷句）。

### 啟用步驟

#### 1. 安裝 Ollama

```bash
brew install ollama
```

#### 2. 下載語言模型

```bash
ollama pull llama3.2
```

#### 3. 啟動 Ollama 服務

```bash
ollama serve
```

#### 4. 在程式碼中啟用

編輯 `ollama_client.py`，將：

```python
OLLAMA_ENABLED: bool = False
```

改為：

```python
OLLAMA_ENABLED: bool = True
```

重新啟動 App，「✨ 潤飾」按鈕會自動變為藍色可用狀態。

### 使用方式

1. 完成一次錄音轉錄
2. 確認結果區有文字後，點擊「✨ 潤飾」
3. 等待 AI 處理（依模型大小而定）
4. 結果區文字會被 AI 潤飾後的版本取代

## 常見問題

### Q：App 啟動後一直顯示「模型載入中…」

**原因**：首次使用需下載模型（`large-v3-turbo` 約 455 MB）。
**解決**：確認網路連線正常，耐心等待；可在終端機觀察下載進度。

---

### Q：點擊錄音後沒有波形，也無法轉錄

**原因**：sounddevice 無法存取麥克風。
**解決**：
1. 確認已安裝 portaudio：`brew install portaudio`
2. 前往**系統設定 → 隱私權與安全性 → 麥克風**，確認 Terminal 已授權
3. 重新啟動 App

---

### Q：全域快捷鍵按下沒有反應

**原因**：缺少 macOS 輔助使用（Accessibility）權限。
**解決**：
1. 前往**系統設定 → 隱私權與安全性 → 輔助使用**
2. 加入並允許 Terminal（或執行 python3 的應用程式）
3. 重新啟動 App

---

### Q：自動貼上沒有作用

**原因**：常見有三種情況。

| 原因 | 解決方式 |
|---|---|
| 輔助使用權限未開啟 | 系統設定 → 隱私權與安全性 → 輔助使用，加入 Terminal |
| 未使用快捷鍵觸發錄音 | 必須用 ⌘⌥R 全域快捷鍵，直接點 App 內按鈕無法捕捉外部視窗 |
| 目標 App 輸入框未聚焦 | 在按下快捷鍵前，先確認目標 App 的輸入框游標已在正確位置 |

---

### Q：設定視窗點擊「重新綁定」後崩潰

此問題已在 v1.0.0 修復。請確認你的版本是最新的：

```bash
git pull origin main
```

---

### Q：轉錄準確度低或語言辨識錯誤

**解決**：
- 在頂部工具列**語言**選單中，手動指定語言（如「中文」），而非「自動偵測」
- 確認使用 `large-v3-turbo` 或更高模型
- 確保錄音環境安靜，說話速度清晰自然

---

### Q：App 報錯「缺少套件」

確認使用虛擬環境執行：

```bash
venv/bin/python3 main.py
```

或重新安裝套件：

```bash
venv/bin/pip install -r requirements.txt
```

---

### Q：如何刪除已下載的 Whisper 模型？

```bash
rm -rf ~/.cache/huggingface/hub/models--Systran--faster-whisper-large-v3-turbo
```

（將最後的模型名稱替換成你想刪除的版本）

---

### Q：如何重置所有設定？

```bash
rm ~/.whisper_app/config.json
```

重新啟動 App 後，所有設定恢復為預設值。
