## 這是什麼

把「語音 → 文字」做到 Speakly 級的桌面使用體驗，但**完全本地**：

- 錄音、語音辨識、AI 潤飾全部在你的 Mac 上完成
- 不需要 API key，不需要網路
- 免費、無訂閱
- 可以處理敏感內容（病歷、律師筆記、面試答辯）

**和 Speakly 的差異**：我們做不到它的 iOS 體驗，但它做不到你的個人字典（雲端不能存你的私密詞彙）。

## 系統需求

| 項目 | 需求 |

| 作業系統 | macOS 12 Monterey 以上（已在 macOS 26.4 驗證） |

| 輔助使用權限 | 全域快捷鍵 + 自動貼上必須 |

## 安裝步驟

### 1. 系統依賴

```bash
brew install portaudio ffmpeg
```

### 2. 虛擬環境

```bash
cd /path/to/project
python3 -m venv venv
source venv/bin/activate
```

### 3. Python 套件

```bash
pip install -r requirements.txt
```

首次安裝約 2-5 分鐘。

### 4.（可選，強烈推薦）Ollama

```bash
brew install ollama
ollama serve                         # 啟動服務（保持這個終端機開著，或放背景）
ollama pull qwen2.5:3b-instruct      # 下載推薦模型（~2 GB）
```

只裝 Ollama 不啟動沒用，要 `ollama serve` 一直在跑才行。建議設成開機自動啟動。

## 啟動 App

```bash
venv/bin/python3 main.py
```

首次啟動：

1. 自動在背景下載 Whisper 模型（預設 `large-v3-turbo`，約 455 MB）。狀態列顯示「模型載入中…」。
2. 下載完、載入完 → 綠點 + 「就緒」→ 可開始錄音。
3. 出現「需要輔助使用權限」彈窗 → 按指示開啟（見[全域快捷鍵](#全域快捷鍵)）。
4. 若有裝 Ollama，啟動 2 秒後會自動探測一次，「潤飾」按鈕變成青色可用。

## 核心流程：按一次、搞定

```
  1. 點到你要輸入文字的地方（Mail、Notion、Slack、Cursor...）
  2. 按住 ⌘⌥R
  3. 說話（中文、英文、中英混都可以）
  4. 放開 ⌘⌥R
         │
         ▼
     Whisper 轉錄
         │
         ▼  ┌───────────────────────────────────┐
     ┌───┤  Ollama 自動潤飾（依前景 app 切 preset）│
     │   │  • Mail → email 風格（含招呼語）        │
     │   │  • Slack / Line → 口語短句              │
     │   │  • Notion → 結構化筆記                 │
     │   │  • Cursor / Xcode → 技術註解            │
     │   │  • 其他 → 通用潤飾                      │
     │   │  + 個人字典逐字保留                     │
     │   └───────────────────────────────────┘
     │
     ▼
  自動 ⌘V 貼到游標位置
```

典型耗時：中英混 5 秒錄音 → 貼上完成約 2-4 秒（看 Ollama 模型大小）。

## 全域快捷鍵

### 預設

**⌘ + ⌥ + R**（Command + Option + R）

按住錄音、放開停止。

### 輔助使用權限

全域快捷鍵 + 自動貼上都需要此權限。

**開啟**：
1. 系統設定 → 隱私權與安全性 → 輔助使用
2. 點 `+` 加入 **Terminal**（或執行此 App 的 python3）
3. 切換右側開關為**開啟**
4. **重新啟動 App**

若沒開啟，App 內點擊按鈕錄音仍可用，但全域熱鍵 + 自動貼上失效。

### 自訂熱鍵

設定 → 快捷鍵 → 重新綁定

**操作**：
1. 對話框跳出後，直接按你想要的組合
2. 即時顯示你按住的組合
3. 鬆開任一鍵 → 鎖定
4. 若只按了修飾鍵會顯示「← 需要加字母/數字/空白」提示
5. 想改主意？直接按新組合會自動重置
6. 點「確認套用」→ 回設定面板 → 點「儲存」

> **macOS 26.4+ 相容**：舊版 pynput 在此對話框會閃退。v2.1 已改用 Tk 原生鍵盤事件，不再炸。

## 自動貼上

說話 → 轉錄 → 自動貼回原 App（無需切視窗或 ⌘C / ⌘V）。

### 開關

動作列「⌨ 自動貼上」按鈕：
- **紫藍色** = 啟用
- **灰色** = 停用

### 運作順序（策略 B：等潤飾完成才貼）

```
1. 切到目標 App（Mail、Slack、Notion…）游標點在輸入框
2. 按住 ⌘⌥R
       │  背景執行緒 osascript → 記下前景 app 名稱
       ▼
3. 說話
4. 放開 ⌘⌥R
       │  Whisper 轉錄
       ▼
5. Ollama 潤飾（若啟用）
       │
       ▼
6. 自動切回目標 App + 模擬 ⌘V
```

### 常見坑

| 狀況 | 原因 | 解法 |
|---|---|---|
| 沒貼上 | 輔助使用權限沒開 | 系統設定開啟 |
| 目標不對 | 沒用熱鍵錄音（點 App 內按鈕不會抓前景 app） | 一律用 ⌘⌥R |
| 貼到錯地方 | 目標 App 輸入框沒聚焦 | 按熱鍵前先確認游標位置 |
| 長錄音等太久 | 大模型潤飾慢 | 改用 `qwen2.5:3b-instruct` |

## AI 潤飾（Ollama）

> **v2.1 起**：Ollama 潤飾是**預設功能**而非進階選項，不再需要改程式碼。

### 為什麼要用

Whisper 原始輸出：
> 嗯那個我今天去那個超市買了一些菜然後就回家了

Ollama 潤飾後：
> 我今天去超市買了一些菜，然後就回家了。

**做了**：
- 刪語氣詞、repetition
- 修同音錯字（在／再、做／坐）
- 補標點斷句
- 保留英文原文（不翻譯 meeting、Kubernetes 等）
- 保留語意（不擅自增刪）

### 啟用步驟

1. **確認 Ollama 服務跑著**：
   ```bash
   curl http://localhost:11434/api/tags    # 應該回 JSON
   ```
   沒跑？`ollama serve`。

2. **確認有模型**：
   ```bash
   ollama list
   ```
   建議：
   - **16 GB Mac** → `ollama pull qwen2.5:3b-instruct`（~2 GB、潤飾 < 1 秒）
   - **32 GB Mac** → `ollama pull qwen2.5:7b-instruct`（~5 GB、1-2 秒、品質更好）

3. **在 App 中啟用**：
   - 設定 → AI 潤飾 (Ollama)
   - 開啟「啟用 AI 潤飾」開關
   - 確認「模型名稱」（可自訂，預設 `qwen2.5:3b-instruct`）
   - 點「測試連線」看綠色 ✓
   - 儲存

4. 動作列的「潤飾」按鈕應該變青色，代表可用。

### 執行流程

每次放開 ⌘⌥R：
```
Whisper 轉錄   ├──► 文字
                 │
                 ▼
            顯示原文 (textbox)
                 │
                 ▼
            背景執行緒：Ollama 潤飾
                 │
                 ▼
            replace textbox 最新一段 → 潤飾版
            + 自動貼上（若啟用）
            + 寫一筆到 polish_log.jsonl
```

**失敗一律降級**：Ollama 掉線、逾時、回空字串→ App 不崩、直接用原文貼上。

### 潤飾按鈕三態

| 外觀 | 意思 |
|---|---|
| 灰色「潤飾」 | 未在設定中啟用 |
| 琥珀色「潤飾（離線）」 | 已啟用但服務不可達（`ollama serve` 沒跑？） |
| 青色「潤飾」 | 可用 |

手動點「潤飾」按鈕 = 對 textbox 內容重跑一次潤飾（會整段取代）。

### 潤飾紀錄

每次潤飾結果（含耗時、長度、preset）寫入：
```
~/.whisper_app/polish_log.jsonl
```

範例：
```json
{"ts":"2026-04-22T12:05:19","model":"qwen2.5:3b-instruct","preset":"email","elapsed_s":3.375,"len_in":44,"len_out":49,"error":null}
```

追蹤自己的使用分佈 / 除錯用。不需要時可在設定關閉。

## 情境 preset（Phase 2 新）

**一句話**：在 Mail 錄音用 email 風格、在 Slack 用口語、在 Cursor 用技術註解，**自動切換**。

### 8 個 preset（含 Phase 3.1 三個 action preset）

#### Target preset（依 app 自動切）

| preset | 會被什麼觸發 | 輸出風格 |
|---|---|---|
| **default**（通用） | 找不到其他 preset 時 | 一般潤飾（去 filler / 修錯 / 補標點） |
| **email** | Mail / Outlook / 郵件 / Superhuman…，或關鍵字 `寄信給 / email to` | 完整句、分段、含招呼語 |
| **chat** | Slack / Discord / 訊息 / Line / WhatsApp…，或關鍵字 `傳訊息給 / dm` | 口語短句、少標點 |
| **note** | Notion / Obsidian / Bear / 備忘錄…，或關鍵字 `筆記 / note mode` | 結構化、可條列 |
| **code_comment** | Xcode / VS Code / Cursor / JetBrains…，或關鍵字 `// / code comment` | 簡潔技術風、名詞保留英文 |

#### Action preset（只靠關鍵字，與前景 app 無關）— Phase 3.1

| preset | 關鍵字 | 輸出 |
|---|---|---|
| **translate_en**（翻英文） | `翻譯英文 / 翻成英文 / translate to english / in english` | 把中文原文翻成自然英文，專有名詞保留原拼寫 |
| **list**（條列） | `條列 / 列點 / 列出 / list mode / bullet points` | 每項一行、開頭 `- ` |
| **meeting_notes**（會議紀錄） | `會議紀錄 / 會議記錄 / 會議摘要 / meeting notes / meeting minutes` | `## 重點` / `## 行動項` / `## 決議` 分段結構 |

**用法範例**：
- 「翻譯英文，我明天下午三點要跟 Jerry 開會討論 Q2 roadmap」
  → `I have a meeting with Jerry tomorrow at 3 PM to discuss the Q2 roadmap.`
- 「條列 買牛奶買麵包買雞蛋」
  → `- 牛奶`<br>`- 麵包`<br>`- 雞蛋`
- 「會議紀錄，今天討論 Q2 roadmap，API 給 Jerry、UI 給 Alice」
  → `## 重點`<br>`- 討論 Q2 roadmap`<br>`## 行動項`<br>`- [Jerry] 負責 API`<br>`- [Alice] 負責 UI`

**關鍵字一律被剝除**再送 LLM，所以輸出不會重複「翻譯英文」四個字。

### 路由規則（由上而下第一個命中生效）

```
  1. 關鍵字開頭匹配（voice shortcut）
     「寄信給 Jerry 明天開會」
        ↓
     命中 email，把「寄信給 」剝掉，送 LLM 的是「Jerry 明天開會」

  2. 前景 app 匹配（小寫 + 去 .app + 支援本地化名）
     Mail / 郵件 / Outlook → email

  3. 都沒命中 → default
```

**關鍵字觸發需要嚴格邊界**：`寄信給Jerry`（沒空格）不觸發，避免誤判。

### 標題顯示

非 default preset 時標題會顯示：
```
轉錄結果  (15s · ZH · large-v3-turbo)  · Email  · 已潤飾 · 1.2s
```

潤飾中 / 已潤飾 / 失敗三種狀態切換時，**preset 名稱永遠保留**。

### 啟用與個別停用

設定 → 情境路由 (Phase 2)：
- **啟用情境自動切換**（主開關，預設開）
- 下面每個 preset 一個 switch，個別停用後改走 default

> **預設都啟用**；如果你覺得 Slack 的口語風格反而不好用，可以只停用 chat。

### 對 auto-paste 的影響

**v2.1 起**：preset 路由**獨立於 auto-paste 開關**。

以前：關掉自動貼上 → preset 也失效（漏掉 app 資訊）
現在：兩者拆開，關掉自動貼上仍會走 preset 路由

## 個人字典（差異化核心）

> Speakly 雲端做不到。你能做到。

### 動機

Whisper 有時會把專有名詞認錯：
- 「cyberpunk」→「瑟柏龐克」
- 「Kubernetes」→「庫貝內茨」
- 你老闆的名字 → 某個同音字

把這些詞放進字典，Whisper **和** Ollama 都會知道「這些字要原樣保留」。

### 檔案位置

```
~/.whisper_app/dictionary.json
```

首次啟動 App 會自動建立範例檔。

### 格式

```json
{
  "terms": [
    {"term": "Whisper Pro", "note": "專案名稱，不要翻譯"},
    {"term": "Kubernetes", "note": "K8s 原文"},
    {"term": "cyberpunk"},
    "簡單字串也可以"
  ],
  "_comment": "每行一個 term 物件；note 可選"
}
```

### 編輯方法

**在 App 內**：設定 → 個人字典 → 「用預設編輯器開啟」→ 編輯 → 存檔 → 回 App 點「儲存」

**直接編輯**：用 VS Code / Cursor 開 JSON 檔、存檔、**在 App 的設定中按「儲存」觸發重新載入**。或重啟 App。

### 注入位置

| 位置 | 上限 | 作用 |
|---|---|---|
| Whisper `initial_prompt` | 前 30 個詞 | 提示模型常用詞彙 |
| Ollama 潤飾 prompt | 前 50 個詞 | 約束 LLM 逐字保留 |

### 實測驗證

**Input**：`我想用 瑟柏龐克 風格設計這個介面`（Whisper 原始錯字輸出）
**字典**：`["cyberpunk"]`
**Output**：`我想用 cyberpunk 風格設計這個介面。`

## 原文／潤飾切換

結果卡標題旁的小分段按鈕：

```
[原文][潤飾]   [✕ 清除]
 灰色  青色高亮
```

- **潤飾版**（預設顯示）：Ollama 處理過的結果
- **原文**：Whisper 原始輸出（**含被 preset 剝除的關鍵字**）

例子：
1. 你說：「寄信給 Jerry 明天下午 3 點」
2. Preset 剝掉「寄信給 」→ LLM 收到「Jerry 明天下午 3 點」
3. LLM 回：「Hi Jerry,\n\n明天下午 3 點…」
4. 畫面預設顯示潤飾版
5. **點「原文」**→ 看到你**完整講的那句**：「寄信給 Jerry 明天下午 3 點」

### 兩顆都灰是什麼意思

- 目前沒有成功潤飾版（Ollama 未啟用／離線／失敗）
- 或 textbox 是空的（還沒錄音過）

### 自己改過文字後

如果你手動編輯了 textbox，再按切換時沉默放棄 → 不會把你的編輯蓋掉。

## 進階：Prompt 熱重載

給開發者用的。

**場景**：你在調整 `prompts.py` 或 `presets.py` 的 prompt，想一次迭代一次試。

**做法**：
1. 設定 → 情境路由 → 確認「Prompt 熱重載」開啟
2. 編輯 `prompts.py`（例如改 `OLLAMA_EMAIL_PROMPT`）
3. 存檔
4. 2 秒內 App 右下角彈出 toast「已重新載入 prompts.py」
5. **下一次錄音就用新 prompt**，不需重啟 App

或手動觸發：設定 → 點「立即重新載入」按鈕。

### 限制

Python 3.13 的 source loader 有**亞秒級 mtime 快取**：檔案在 1 秒內被改兩次會用舊 bytecode。實務上沒人會這樣操作，但自動化測試要注意。

## 進階：Eval runner

給 prompt 工程師用的。

**目的**：客觀比對不同 prompt 在一組固定音訊上的品質。

### 準備 golden set

```
tests/golden_set/
├── 001_short_chinese.wav             (16 kHz mono)
├── 001_short_chinese.expected.txt    (你手寫的理想潤飾結果)
├── 001_short_chinese.meta.json       (選填：{"preset": "email"})
├── 002_mixed_lang.wav
├── 002_mixed_lang.expected.txt
└── ...
```

建議 10-15 條，涵蓋短句／長句／純中文／中英混／filler 密集／專有名詞／各 preset 情境。

### 跑

```bash
venv/bin/python3 eval_runner.py                     # 全跑
venv/bin/python3 eval_runner.py --id 001            # 指定 ID
venv/bin/python3 eval_runner.py --no-polish         # 只跑 Whisper
venv/bin/python3 eval_runner.py --model qwen2.5:7b  # 換模型
```

### 產出

`tests/reports/{timestamp}.csv` — 每列一個 case，欄位：

| 欄位 | 意義 |
|---|---|
| `raw_text` | Whisper 原始輸出 |
| `polished_text` | Ollama 潤飾 |
| `expected_text` | 你寫的理想版 |
| `whisper_ms` / `polish_ms` | 耗時 |
| `english_preserved_ratio` | 英文單字保留率（0-1） |
| `filler_removed_estimate` | filler 刪除估計次數 |
| `matches_expected` | 完全相符？ |

Console 印彙總統計。

## 模型選擇指南

### Whisper（語音辨識）

| 模型 | 大小 | 速度 | 精準度 | 建議 |
|---|---|---|---|---|
| `tiny` | 39 MB | ⚡⚡⚡⚡⚡ | ★★ | 英文速記測試 |
| `base` | 74 MB | ⚡⚡⚡⚡ | ★★★ | 輕量日常 |
| `small` | 244 MB | ⚡⚡⚡ | ★★★★ | 中文效果佳 |
| `medium` | 769 MB | ⚡⚡ | ★★★★★ | 高精準度 |
| **`large-v3-turbo`** | 809 MB | ⚡⚡⚡ | ★★★★★ | **預設**，中英混最佳，Metal GPU |
| `large-v3` | 1.55 GB | ⚡ | ★★★★★ | 最高精準度，16 GB+ |

| `qwen2.5:7b-instruct` | 7B | ~5 GB | 1-2s | 32 GB Mac 首選 |
| `qwen2.5:14b-instruct` | 14B | ~9 GB | 3-4s | 48 GB+ |
| `gemma3:12b` | 12B | ~8 GB | 2-3s | 若有 pulled 也可用 |

## 常見問題

### Q：模型一直顯示「載入中」

首次使用需下載 455 MB，確認網路。終端機可以看下載進度。

### Q：按下熱鍵沒反應

90% 是輔助使用權限問題。系統設定 → 隱私權 → 輔助使用 → 加入 Terminal → 重啟 App。

### Q：自動貼上沒作用

三種可能，按機率排：
1. 輔助使用權限沒開
2. 沒用熱鍵錄音（點 App 內按鈕無法抓前景）
3. 目標 App 輸入框沒聚焦

### Q：「潤飾（離線）」一直不變青色

Ollama 服務沒跑。開終端機 `ollama serve`，或用下面驗證：
```bash
curl http://localhost:11434/api/tags
```

### Q：Ollama 明明跑著，連線還是失敗

檢查「服務位址」欄位是不是 `http://localhost:11434`。有些 Ollama 版本會綁 127.0.0.1 而不是 localhost，試試 `http://127.0.0.1:11434`。

### Q：潤飾後的 email 亂加了我沒講的簽名檔

Email preset 有明確指示「不要加簽名檔」，但 LLM 偶爾會不聽話。你可以：
- 點「原文」看原始輸出
- 或去 `prompts.py` 把 `OLLAMA_EMAIL_PROMPT` 的約束強化
- 或換更大的模型（7B / 14B）

### Q：我的專有名詞一直被認錯

去編輯字典：設定 → 個人字典 → 「用預設編輯器開啟」→ 加條目。

### Q：重新綁定對話框閃退 / 卡住

v2.1 已修（macOS 26.4+ TSM 主執行緒斷言）。`git pull && git log --oneline` 確認有 `0806c8c` 這個 commit。

### Q：轉錄準確度低

- 頂部選單手動指定語言（如「中文」），別用「自動偵測」
- 用 `large-v3-turbo` 或更高
- 環境噪音低、講話清楚

### Q：如何重置所有設定

```bash
rm ~/.whisper_app/config.json
```

重啟 App 恢復預設。字典和 polish_log 不會被動到。

### Q：怎麼看自己用了多少次潤飾

```bash
wc -l ~/.whisper_app/polish_log.jsonl        # 總次數
jq -s '[.[].elapsed_s] | add / length' ~/.whisper_app/polish_log.jsonl  # 平均耗時
```
