## 安裝步驟

### 步驟一：安裝系統依賴

開啟終端機，執行：

```bash
brew install portaudio ffmpeg
```

- **portaudio**：麥克風錄音驅動
- **ffmpeg**：音訊處理工具

### 步驟二：建立虛擬環境

```bash
cd /Users/jerrychen/project/Claude_code
python3 -m venv venv
source venv/bin/activate
```

### 步驟三：安裝 Python 套件

```bash
pip install customtkinter==5.2.2 sounddevice==0.5.1 "numpy<3" \
            faster-whisper pyperclip==1.9.0 pynput==1.7.7 requests==2.32.3
```

> 初次安裝約需 2–5 分鐘（依網速而定）。

## 啟動 App

```bash
cd /Users/jerrychen/project/Claude_code
venv/bin/python3 main.py
```

或若已啟動虛擬環境：

```bash
python3 main.py
```

**首次啟動注意事項：**

1. App 會自動在背景下載 Whisper 模型（預設 `base`，約 145 MB），過程中狀態列顯示「模型載入中…」。
2. 模型下載完成後，狀態列變為「模型已就緒 (base)」，即可開始錄音。
3. 若出現「需要輔助使用權限」彈窗，請依指示開啟 macOS 權限（詳見[全域快捷鍵](#全域快捷鍵)章節）。

## 錄音操作

### 方式一：點擊按鈕

| 步驟 | 操作 | 按鈕狀態 |
|---|---|---|
| 1 | 點擊綠色「🎤 點擊錄音」按鈕 | 變為紅色脈衝「● 正在錄音…」|
| 2 | 對麥克風說話 | 波形動畫即時更新，計時器顯示錄音時長 |
| 3 | 點擊紅色按鈕停止 | 變為橘色「⠋ 轉錄中…」|
| 4 | 等待轉錄完成 | 結果顯示在文字區，右下角顯示完成通知 |

### 方式二：全域快捷鍵（Push-to-Talk）

**按住** ⌘⇧Space → 說話 → **放開** → 自動轉錄

> 此方式可在任何 App 前景下使用，無需切換到此視窗。

### 按鈕顏色說明

| 顏色 | 狀態 | 說明 |
|---|---|---|
| 綠色 | 待機（IDLE）| 就緒，可開始錄音 |
| 紅色脈衝 | 錄音中（RECORDING）| 正在收音，計時器運行中 |
| 橘色 | 轉錄中（PROCESSING）| Whisper 模型分析音訊，請稍候 |

## 全域快捷鍵

### 預設快捷鍵

**⌘ + ⇧ + Space**（Command + Shift + 空白鍵）

### 操作方式

- **按住** 快捷鍵 → 開始錄音
- **放開** 快捷鍵 → 自動停止並開始轉錄

### macOS 輔助使用權限

全域快捷鍵需要 macOS「輔助使用（Accessibility）」權限，才能在其他 App 前景時監聽按鍵。

**開啟步驟：**

1. 點擊 App 啟動時出現的引導視窗中的「開啟系統設定」
2. 或手動前往：**系統設定 → 隱私權與安全性 → 輔助使用**
3. 點擊「+」加入 **Terminal**（或執行 python3 的應用程式）
4. 確認切換開關為開啟狀態
5. **重新啟動** App

> 若跳過此步驟，仍可在 App 視窗中使用按鈕錄音，全域快捷鍵功能將無效。

### 自訂快捷鍵

1. 點擊「⚙ 設定」→「快捷鍵」區塊
2. 點擊「重新綁定」
3. 在彈出視窗中**按下**你想要的按鍵組合
4. 確認顯示正確後，點擊「確認套用」
5. 點擊「儲存」

## 轉錄結果操作

### 複製到剪貼簿

點擊「📋 複製」→ 全部文字複製到系統剪貼簿 → 右下角顯示「已複製到剪貼簿」通知。

### 儲存為文字檔

點擊「💾 存檔」→ 選擇儲存路徑與檔名 → 存成 `.txt` 格式。

### 清除結果

點擊「清除 ✕」→ 清空轉錄文字區，重新開始。

### 多次錄音追加

預設開啟「追加模式」：每次新的錄音結果會**附加**到現有文字下方，並以分隔線區分。

可在設定中關閉此選項，改為每次覆蓋。

### 結果標題資訊

轉錄完成後，結果區標題會顯示：

```
📝 轉錄結果  (15s · ZH · base)
```

- `15s` — 錄音時長
- `ZH` — 偵測到的語言（ZH=中文、EN=英文 等）
- `base` — 使用的模型大小

## 設定面板

點擊「⚙ 設定」開啟設定視窗。

### 語音辨識

| 設定 | 說明 |
|---|---|
| 模型大小 | 選擇 Whisper 模型（詳見[模型選擇指南](#模型選擇指南)）|
| 辨識語言 | 設定語言或選擇「自動偵測」|

### 快捷鍵

| 設定 | 說明 |
|---|---|
| 全域快捷鍵 | 顯示目前設定的按鍵組合 |
| 重新綁定 | 點擊後進入快捷鍵偵測模式，按下新組合即可 |

### 輸出

| 設定 | 預設 | 說明 |
|---|---|---|
| 每次錄音追加到結果 | 開啟 | 開啟：新結果附加到下方；關閉：每次覆蓋 |
| 轉錄後自動複製 | 關閉 | 開啟後，每次轉錄完成自動複製文字到剪貼簿 |

### 關於

顯示 Whisper 模型快取位置與設定檔路徑，可點擊「開啟設定資料夾」直接在 Finder 開啟。

- **設定檔位置**：`~/.whisper_app/config.json`
- **模型快取位置**：`~/.cache/huggingface/hub/`

## 模型選擇指南

| 模型 | 大小 | 速度 | 精準度 | 建議用途 |
|---|---|---|---|---|
| `tiny` | 39M | ⚡⚡⚡⚡⚡ | ★★☆☆☆ | 快速英文速記、測試用 |
| `base` | 74M | ⚡⚡⚡⚡ | ★★★☆☆ | **日常使用推薦**，速度與精準度平衡 |
| `small` | 244M | ⚡⚡⚡ | ★★★★☆ | 中文辨識效果更佳，可接受稍慢速度 |
| `medium` | 769M | ⚡⚡ | ★★★★★ | 高精準度，需 8GB+ 記憶體 |
| `large` | 1.5G | ⚡ | ★★★★★ | 最高精準度，需 16GB+ 記憶體，速度慢 |

**切換模型：**
- 在頂部工具列下拉選單直接選擇，或進入設定面板
- 切換後首次轉錄會自動下載並載入新模型（需等待）

**中文辨識建議：** `small` 或以上的模型中文辨識效果明顯優於 `tiny` / `base`。

## Ollama 整合（進階）

Ollama 功能可讓 AI 大語言模型自動潤飾轉錄文字（修正錯字、調整語氣、重新斷句）。

### 啟用步驟

#### 1. 安裝 Ollama

前往 [https://ollama.com](https://ollama.com) 下載安裝，或使用 Homebrew：

```bash
brew install ollama
```

#### 2. 下載語言模型

```bash
ollama pull llama3.2
```

#### 3. 啟動 Ollama 服務

```bash
ollama serve
```

#### 4. 啟用 App 內的 Ollama 功能

編輯 `ollama_client.py`，將第 19 行：

```python
OLLAMA_ENABLED: bool = False
```

改為：

```python
OLLAMA_ENABLED: bool = True
```

重新啟動 App，「✨ Ollama 潤飾」按鈕會自動變為可用狀態。

### 使用方式

1. 完成一次錄音轉錄
2. 確認結果區有文字後，點擊「✨ Ollama 潤飾」
3. 等待 AI 處理（依模型大小與文字長度而定）
4. 結果區文字會被 AI 潤飾後的版本取代

## 常見問題

### Q：App 啟動後一直顯示「模型載入中…」

**原因**：首次使用需要下載 Whisper 模型（base 約 145 MB）。
**解決**：確認網路連線正常，耐心等待。可在終端機查看下載進度。

---

### Q：點擊錄音後沒有聲音波形，也沒辦法轉錄

**原因**：sounddevice 無法存取麥克風，或 portaudio 安裝有問題。
**解決**：
1. 確認已安裝 portaudio：`brew install portaudio`
2. 前往**系統設定 → 隱私權與安全性 → 麥克風**，確認 Terminal 有麥克風權限
3. 重新啟動 App

---

### Q：全域快捷鍵按下去沒有反應

**原因**：缺少 macOS 輔助使用（Accessibility）權限。
**解決**：
1. 前往**系統設定 → 隱私權與安全性 → 輔助使用**
2. 加入並允許 Terminal（或執行 python3 的應用程式）
3. 重新啟動 App

---

### Q：轉錄結果語言不正確或準確度低

**解決**：
- 在頂部工具列**語言**下拉選單，手動指定語言（如「中文」）而非「自動偵測」
- 切換到較大的模型（如 `small` 或 `medium`）
- 確保錄音環境安靜，說話速度清晰

---

### Q：App 報錯「缺少套件」

**解決**：確認使用正確的虛擬環境執行：

```bash
venv/bin/python3 main.py
```

或重新安裝套件：

```bash
venv/bin/pip install -r requirements.txt
```

---

### Q：如何刪除已下載的 Whisper 模型？

模型快取位於 `~/.cache/huggingface/hub/`，可手動刪除：

```bash
rm -rf ~/.cache/huggingface/hub/models--Systran--faster-whisper-base
```

（將 `base` 替換為你想刪除的模型名稱）

---

### Q：設定檔在哪裡？如何重置設定？

設定檔位於 `~/.whisper_app/config.json`，刪除即可重置為預設值：

```bash
rm ~/.whisper_app/config.json
```
