- **日期**：2026-05-23

- **前一版**：v2.5.0（2026-05-23）

## TL;DR

新增淺色主題、Settings 內可切換。配色是 **Hybrid（Apple 結構 + Claude 溫度）** ── 微暖白 `#FAFAF7` 背景、純白 `#FFFFFF` 卡片、Claude 珊瑚 `#D97757` 主 CTA、Apple 藍 `#007AFF` 作為連結輔助。切換用 Settings 點 chip → confirm 對話框 → 自動重啟 App（~2 秒）。暗色主題（v2.5.0 zinc + cyan）維持不動。

## 主要改進

### 🎨 淺色主題 — Variant C Hybrid

設計理念：**蘋果 SF Pro hierarchy 的克制 + Claude 品牌色點睛**。背景比純白略暖、子意識舒服；CTA 用 Claude 珊瑚一目了然；連結 / info 用 Apple 系統藍語意清楚。

**色彩規格**（淺色主題 token）：

| Token | Value | 用途 |
|---|---|---|
| `BG` | `#FAFAF7` | 視窗背景（微暖白） |
| `SURF_1` | `#FFFFFF` | 卡片主體（Apple-clean） |
| `SURF_2` | `#F1F0EC` | 副表面 / raised |
| `SURF_3` | `#E5E3DC` | pressed / hover |
| `SURF_4` | `#D4D2C8` | 邊框、分隔線 |
| `TEXT_1` | `#1A1612` | 標題（warm 近黑、AAA） |
| `TEXT_2` | `#3D362B` | 內文（warm dark） |
| `TEXT_3` | `#6E6E73` | 輔助（Apple 冷灰、刻意冷平衡暖度） |
| `TEXT_4` | `#A8A8AD` | 停用 |
| `ACCENT` | `#D97757` | Claude coral（主 CTA） |
| `ACCENT_HV` | `#C66445` | 珊瑚 hover |
| `ACCENT_BG` | `#F6E8DE` | 珊瑚 chip 淡底 |
| `LINK` | `#007AFF` | Apple 藍（連結 / info） |
| `SUCCESS` | `#2E8B57` | balanced green |
| `DANGER` | `#D14B41` | warm Apple red |
| `WARN` | `#C7842B` | muted amber |
| `INDIGO` | `#6366F1` | 自動貼上 |

WCAG AA 已驗證：TEXT_1/2 對 BG/SURF_1 均 ≥10:1（AAA）、TEXT_3 ≥4.5:1（AA body）、ACCENT 對白 ≥3:1（AA Large/UI）。

### 🎛 Settings 內「外觀」section

Settings 視窗第一個 section 就是「外觀」，兩顆 segmented chip（深色 / 淺色）。點 chip：
- 跟現在主題不同 → 彈 confirm modal（含「進行中錄音 / 潤飾 會被中斷」警告）
- 點「確認、重啟」→ 立刻 save config + toast「主題切換中、~2 秒」+ 800ms 後 cleanup + spawn + exit
- 點「取消」或 ESC → 視覺還原、不存檔

### 🔁 Restart auto-relaunch UX

**為什麼不做 live switch**：Ambient Chamber 是手繪 Canvas、`tokens.py` 被 30+ 處 import 為 single-value 常數、`MiniRecordingWindow` 用 PyObjC 設過 NSWindow level。要做 live switch 需要 ~300 LOC + 多輪迭代修「漏改的 widget」bug。Restart 路徑 ~100 LOC 乾淨可靠、~2 秒體驗。

**Restart 順序（cleanup-first，避免 race）**：

```
1. SettingsWindow.cfg.save() —— config.json 落地新 theme
2. SettingsWindow.destroy() —— 設定視窗關閉
3. AppWindow._do_theme_relaunch()
   ├─ _show_toast("主題切換中、~2 秒…")
   └─ self.after(800, _do_relaunch_sequence)
        ├─ Step A：self.on_close()
        │    ├─ hotkey_mgr.stop()         移除 NSEvent monitor
        │    ├─ recorder.stop()            釋放麥克風
        │    ├─ mini_window.destroy()
        │    ├─ history_store.close()      關 SQLite
        │    └─ Cocoa observers cleanup
        ├─ Step B：main._relaunch_app()
        │    ├─ .app bundle: subprocess.Popen(["open","-n",APP_BUNDLE])
        │    └─ dev mode:   os.execv(sys.executable, [...])
        └─ Step C：sys.exit(0)（.app 路徑）/ execv 已 replace process
```

舊 process 資源完全釋放後才 spawn 新 process — **無雙 NSEvent monitor 衝突 / 無 mic 鎖 race / 無 config.json race**。Eng Review Issue 1 解決。

### 🛡 防呆：未知 theme 值 fallback

使用者實驗性手改 `~/.whisper_app/config.json` 把 `"theme"` 改成 `"purple"` 或舊版 typo（`"Light"` / `"LIGHT"`）→ `_PALETTES[_THEME]` KeyError 啟動 crash。

`tokens.py` 加一行 fallback 保護：

```python
if _THEME not in _PALETTES:
    log.warning(f"tokens: unknown theme {_THEME!r}, fallback to 'dark'")
    _THEME = "dark"
```

靜默降級 + log warning（使用者查 log 能找到原因）。Eng Review Issue 2 解決。

## 架構變化

### tokens.py：module-level 常數 → theme-aware

```python
# Before（v2.5.0）
BG = "#000000"
SURF_1 = "#0E0E10"
...
```

```python
# After（v2.6.0）
_PALETTES = {"dark": {...}, "light": {...}}
_THEME = _active_theme()   # 讀 cfg.theme，未知值 fallback "dark"
_P = _PALETTES[_THEME]
BG = _P["BG"]
SURF_1 = _P["SURF_1"]
...
```

**關鍵特性**：既有 `from tokens import BG, SURF_1, ACCENT` **一行都不用改**、所有 30+ 檔自動跟著 active theme。Module-level 常數在 import 時鎖定，不支援 live switch（換 theme 必須 App restart）。

### 新增 token：`LINK`

連結 / info icon 專用色。深色主題下 = `ACCENT`（cyan）；淺色主題下 = `#007AFF`（Apple 藍）。語意分離：珊瑚 = CTA、藍 = 資訊。

### config.py 新增欄位

```python
theme: str = "dark"   # "dark" | "light"，預設 dark
```

`valid_data` filter 機制讓舊使用者 config.json 無此欄位也能載入（fallback dark）。

### main.py 新增 `_relaunch_app()`

```python
def _relaunch_app() -> bool:
    """重啟自己。呼叫者必須先完成 cleanup 再呼叫本函式（避免 race）。"""
    if is_bundled and _APP_BUNDLE.exists():
        subprocess.Popen(["open", "-n", str(_APP_BUNDLE)], start_new_session=True)
        return True
    os.execv(sys.executable, [sys.executable, str(Path(__file__))])
    return False  # 走到這行表示 execv 失敗
```

從 `.app` bundle 啟動就 `open -n`；dev 模式從 CLI 就 `os.execv`。失敗（罕見）回 False，呼叫者顯示 toast 提示手動處理。

### `ctk.set_appearance_mode` 對齊

`main.py` 在 `Config.load()` 後立刻：
```python
ctk.set_appearance_mode("Light" if cfg.theme == "light" else "Dark")
```

讓 CustomTkinter 內建 widget（scrollbar、dropdown 預設樣式）跟隨 cfg.theme。自訂 widget 的色彩來自 tokens.py 已自動跟著切。

## Migration

升級到 v2.6.0：
1. **重建 .app bundle**：`bash build_app.sh`（從 `_version.py` 自動讀 2.6.0、Info.plist 同步更新）
2. **首次啟動可能要重新授權**（如果 build_app.sh 跑了 `tccutil reset`、4 個權限需重新給）
3. 設定預設 `theme="dark"`、與 v2.5.0 完全一致 → **預設無視覺變化**
4. 進設定視窗 → 第一個 section 「外觀」→ 點「淺色」chip → confirm → ~2 秒後就在新主題

Breaking changes：**無**。所有外部 API（hotkey 字串、設定欄位、log 行為）向後相容。

## 已知限制

- **Live switch（不重啟即時切換）** 延後：Ambient Chamber Canvas + tokens.py module-level 常數 + Mini HUD NSPanel 等都要重做、~300 LOC 風險高。實際使用一週後評估真實需求再決定。
- **System theme follow（跟系統 dark/light 自動切）** 延後：需要 `AppleInterfaceThemeChangedNotification` 監聽 + restart auto-relaunch、~50 LOC 追加。
- **第三主題（high-contrast / colorblind）** 不在範圍。
