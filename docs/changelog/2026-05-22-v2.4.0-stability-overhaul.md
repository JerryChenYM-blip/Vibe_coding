## TL;DR

**這是一次以「不要再閃退、不要再卡死」為核心的根治型大改造。** 把 macOS 26.4+ 上 pynput 1.7.7 造成的 8+ 種 crash 全部砍掉，全面改用 Apple 原生 API（NSEvent / NSApplicationWillBecomeActiveNotification / NSPanel-level）。**架構上跟 Speakly 走同一條路**。

不是新功能版本，是**徹底解決使用體驗的基礎建設版本**。

---

## 主要改進（依使用者感受排序）

### 1. 🎤 全域熱鍵改用單按右 Cmd（tap toggle）

**Before**：`⌘⌥R` 三鍵組合、按住說話、放開停止
**After**：右 Cmd 一下放開＝開始、再按一下放開＝停止。與螢幕錄音按鈕行為一致。

- 支援 lone-modifier 字串格式：`right_cmd` / `right_option` / `right_ctrl` / `right_shift` / `left_*` 同理
- 左右側 modifier 明確區分（`Key.cmd_r` vs `Key.cmd_l`）
- Dead-key 保護：按 Right Option + e 仍能打 é，不會誤觸發錄音

舊設定 `cmd+alt+r` 仍可用，但不會自動遷移，需自己改。

### 2. 💀 不再閃退（macOS 26.4+ TSM / PAC crash 全砍）

**Before**：pynput 1.7.7 在 macOS 26.4+ 平均**每 30 分鐘 crash 一次**：
- `TSMGetInputSourceProperty` 主執行緒斷言（SIGTRAP）
- `SLEventTapIsEnabled` Pointer Authentication 違規
- `dispatch_assert_queue_fail` 各種變體

**After**：用 PyObjC `NSEvent.addGlobalMonitorForEventsMatchingMask_handler_` + local monitor 全面取代 pynput Listener。Handler 永遠在主執行緒、不用 ctypes、不踩 TSM/PAC 任何路徑。

pynput Controller（送 ⌘V）保留 — 已驗證主執行緒呼叫安全。

### 3. 🪟 視窗最小化後叫得回來（5 層保險絲）

**Before**：⌘M 最小化或點視窗縮圖 → 永遠卡在 Dock，必須 force quit 重開
**After**：5 層保險絲確保 user 透過任何方式（Dock icon、視窗縮圖、⌘+Tab、Spotlight、Mission Control）取得焦點時，視窗最慢 500ms 內恢復：

1. `::tk::mac::ReopenApplication` Tk callback
2. `<<Activate>>` Tk virtual event
3. `NSApplicationDidBecomeActiveNotification`（Cocoa）
4. `NSApplicationWillBecomeActiveNotification`（**主力修法**，源自 Apple Dev Forum 706772）
5. `NSApp.isActive` 每 500ms 輪詢 transition False→True

### 4. ⏱ 閒置 1-2 小時後熱鍵不會悄悄失效

**Before**：閒置一段時間後按熱鍵沒反應，無 log、無 crash，只能重啟 App
**After**：
- App Nap 抑制（`NSProcessInfo.beginActivityWithOptions_`）
- Listener watchdog 每 5s 心跳 + 每 10 分鐘無條件 force restart
- NSEvent 本身就不會 idle timeout（這條路徑在 Fix 7 後其實不會發生，watchdog 是 belt-and-suspenders）

### 5. 🎬 錄音時螢幕中下方有 HUD（Speakly 風格）

**Before**：MiniRecordingWindow 預設關閉、位置固定右下角、別人全螢幕時看不到
**After**：
- 預設**打開**
- 位置追**游標所在螢幕中下方**（多螢幕跟你跑）
- PyObjC `NSStatusWindowLevel` + `setCollectionBehavior_`：**跨 Space / 全螢幕可見 / 不搶 focus**
- 跟 Speakly 的「隨時知道在錄音」體驗一致

### 6. 🧪 轉錄不會卡死、不會把第二段錄音踩平

**Before**：
- Whisper inference throw exception → state 卡在 `processing` 永遠回不來
- `_processing_timeout_check` 60s force-idle 後，舊 inference 慢慢回來會把第二段錄音的 UI 全部覆寫

**After**：
- Whisper thread 包 try/except → 失敗時 state 切回 idle + toast「⚠ 轉錄失敗」
- Processing 動態 timeout（`max(60s, audio_duration × 1.0)`）→ 錄 5 分鐘音檔在 CPU backend 不會誤殺
- Stale callback drop：state 已非 `processing` 時 `_on_transcription_done` 自動 drop + log_action

### 7. 🔐 macOS 權限授權「授權一次永遠有效」

**Before**：從不同終端機 / Claude Preview / Spotlight 啟動 → 被歸到不同 responsible process → 每個都要重新授權麥克風、輔助使用、AppleScript
**After**：
- `~/Applications/WhisperPro.app` 薄殼 .app bundle（`build_app.sh` 一鍵建構）
- 內嵌 Python shim（`Contents/Frameworks/Python.app/`，自製 bundle id `com.jerrychen.whisperpro.python`），解決 TCC silent deny（系統 Python.app 沒 `NSMicrophoneUsageDescription`）
- launcher 用 `arch -arm64 exec` 確保 process 樹乾淨、ad-hoc 簽署過 Gatekeeper

授權一次，未來從 Dock / Spotlight / Finder 任何路徑啟動都生效。

## Bug fixes（不在上面 1-7 主題裡的）

- **TSM auto-paste 閃退**：`_do_auto_paste` 從背景 thread 改主執行緒呼叫 pynput Controller（`TSMGetInputSourceProperty` 主執行緒限制）
- **左右 Cmd 同時按住、放開一顆會被誤判**：FlagsChanged 不能只看 `modifierFlags() & cmd_bit`（左右共用 device-independent bit），改用 `_ns_held_modifiers` 集合做 state machine
- **App 啟動視窗按鈕被遮**：`WIN_H=800→880`、`minsize=750→860`（AppWindow 自然 reqheight ≈ 858px）
- **音訊靜音 guard log 缺實際數值**：改成輸出 `peak=X.XXXXXX rms=X.XXXXXX below threshold=0.002`，下次同類問題秒判
- **`AppWindow` 是 CTkFrame 不是 Toplevel**：dock-reopen handler 全部走 `self.winfo_toplevel()` 而非 `self`（之前 5 層保險絲全部踩這個雷靜默 AttributeError）

## 架構變化

### 砍掉的東西

- ❌ pynput Listener（`pynput.keyboard.Listener` 整段移除）
- ❌ pynput callback handler（`_on_p` / `_on_r` / `_on_p_lone` / `_on_r_lone` 透過 NSEvent backend 仍在用，但不再有 pynput thread 來呼叫）
- ❌ `get_event_tap()`（NSEvent 沒這概念）
- ❌ Watchdog Layer 2（CGEventTap 健康度體檢、Fix 6 v2 短暫存在後 Fix 7 砍掉）
- ❌ `_NS_TITLE` class const（改用 `self._ns_title` instance-unique）

### 新增的東西

- ✅ NSEvent backend in `hotkey_manager.py`（`_handle_ns_event` + `_on_ns_event_global` + `_on_ns_event_local`）
- ✅ `_disable_app_nap()` in `main.py`（module-level `_APP_NAP_TOKEN` 強參照）
- ✅ `_install_cocoa_activation_observer` + `_on_cocoa_app_will_activate` in `gui.py`（PyObjC NSNotificationCenter）
- ✅ `_restore_root_if_minimized(source)` helper（5 個 dock-reopen 來源全走這）
- ✅ `_poll_window_visibility` 500ms 輪詢 NSApp.isActive 轉換
- ✅ `MiniRecordingWindow._upgrade_to_panel_level` PyObjC NSPanel-level
- ✅ `MiniRecordingWindow._position_at_cursor_screen_bottom` 多螢幕 y-flip 座標換算
- ✅ `build_app.sh` 薄殼 .app builder（含 shim Python.app）
- ✅ 7 個新 regression test（PR #13 收尾的 code review fix）

### 重要 metric

| 指標 | Before | After |
|---|---|---|
| crash 頻率 | 平均 ~30 分鐘 1 次 | 0（NSEvent backend 無 TSM/PAC 路徑） |
| 閒置後 hotkey 失效 | 1-2 小時必 fail | 多層保險絲，理論不再發生 |
| Dock 最小化恢復 | 100% fail（無回應）| 5 層保險絲、500ms 內恢復 |
| 多螢幕 HUD | 固定右下、別人全螢幕看不到 | 跨 Space + 跟游標螢幕 + 不搶 focus |
| 權限重複授權 | 換 launcher 就重新授權 | 一次永久（透過 .app bundle）|
| 測試覆蓋（穩定性）| 12 個 test | 47 個 test |

## Migration

### 升級到 v2.4.0

1. **重建 .app bundle**：
   ```bash
   cd /Users/jerrychen/project/Claude_code
   bash build_app.sh
   ```

2. **首次啟動可能要重新授權**（因為 shim Python.app 的 bundle id 是新的 `com.jerrychen.whisperpro.python`）。授權後**未來不再要求**。

3. **舊熱鍵設定不會自動遷移**：如果你以前用 `cmd+alt+r`，要自己改成 `right_cmd`：
   ```bash
   python3 -c "import json, pathlib; p=pathlib.Path.home()/'.whisper_app'/'config.json'; d=json.loads(p.read_text()); d['hotkey']='right_cmd'; p.write_text(json.dumps(d, ensure_ascii=False, indent=2))"
   ```

4. **Mini HUD 預設打開**。如果你不想要：設定面板裡可以關。

### Breaking changes

無。所有外部 API（hotkey 字串格式、設定欄位、log 行為）都向後相容。
